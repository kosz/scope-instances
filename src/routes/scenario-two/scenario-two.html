2: Using Services for sharing functionality between Controllers

It's considered a good idea to separate concerns from the Controller, address them in a Service, then provide that service to be used by more than one controllers, or even other services.

The current scenario, could be a candidate, once it's unusual requirement is investigated further. If it turns out that the value needs to be shared with even other angular objects, the value can be stored in a service, which would then be available for injection wherever it's required.
