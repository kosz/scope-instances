1: Keeping things as simple as possible

code on github

It's sometimes good to stop and think if things are not needlesly overcomplicated, especially for the purpose of code maintanability. For the question at hand I feel the need to ask :

Do we really need two controllers for such a simple operation ?

If the answer is no, then we can just eliminate one of the controllers. 

angular.module('myApp')
  .controller('ParentCtrl', function($scope) {
    $scope.value = 42;
  });
      
<div ng-app="myApp" ng-controller="ParentCtrl">
  <input ng-model="value" type="text" />
  {{value}}
</div>

Note that while this code looks very much in tune with angular documentation code, the other solutions are better suited for real world angular apps.
